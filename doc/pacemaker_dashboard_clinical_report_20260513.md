# Pacemaker Dashboard 临床汇报研究报告

生成日期：2026-05-13  
目标用途：作为临床医生介绍型 PPT 的唯一内容底稿  
隐私边界：仅使用聚合数据；不展示患者姓名、登记号、单个患者 JSON、原始 Excel 路径或原始报告截图。

## 1. 一句话结论

Pacemaker Dashboard 的核心价值，是把散落在文件夹和 Excel 中的起搏器程控随访报告，整理成医生可以按患者查看、按时间复核、按字段复用的本地患者资料库。

这套系统不替代医生判断，也不输出诊断建议。它承担的是资料整理、字段提取、纵向归档、查询展示和回源复核的工作，目的是把医生从重复翻文件、找历史、核对参数的低效环节中解放出来。

## 2. 项目背景与临床痛点

程控随访资料的临床源头仍然是不同品牌、不同年份、不同模板的 Excel 报告。单份报告在当次随访中可读，但当资料积累到数百名患者、上千条记录后，问题会集中暴露：

- 找患者慢：需要在文件夹、文件名和 Excel 工作表之间来回查找。
- 看历史难：同一患者多次程控记录不天然聚合，纵向变化需要人工拼接。
- 品牌模板不统一：美敦力、雅培、百多力、波士顿科学、创领、Vitatron 等报告字段和表格结构不同。
- 复核链条长：关键参数、事件记录、结论和签名日期等信息需要回到原报告确认。
- 科研复用成本高：没有结构化字段和患者级档案时，队列筛选和统计分析前处理负担很重。

因此，这个项目首先解决的是资料组织方式问题：从“按文件存在”转向“按患者组织”。

## 4. 系统工作流

项目的数据流可以概括为六步：

1. 原始报告进入：原始 Excel 程控报告保留为临床源头。
2. 模板匹配：根据文件名、品牌和设备报告类型匹配模板。
3. 字段提取：读取 header、基础设置、测量参数、电池导线、阈值测试、事件记录、签名日期等区域。
4. 患者合并：把同一患者的多次程控记录聚合到患者级 JSON 档案。
5. 数据打包：生成前端可读取的 data_bundle.js、索引和记录文件。
6. 看板展示：医生通过本地网页查看群体概况、患者详情和随访历史。

当前模板库包含 14 个模板条目，覆盖品牌：传导束起搏 / 创领 / 波科 / 百多力 / 美敦力 / 美敦力Micra AV / 雅培；报告类型包括：CRT-D报告单 / CRT-P报告单 / EV-ICD报告单 / ICD报告单 / 起搏器报告单。

## 5. 医生使用场景

临床汇报中应把使用方式讲成医生熟悉的动作，而不是讲代码模块。

- 找患者：从患者索引、搜索入口或列表进入目标患者档案。
- 看设备身份：先确认品牌、型号、植入日期、设备类型等基础信息。
- 看设备状态：查看电池寿命、导线/阈值、感知、模式和频率设置。
- 看事件与结论：集中查看事件记录、建议下次程控时间、报告结论和备注。
- 看随访历史：同一患者多次记录按时间排序，便于比较变化。
- 回原报告复核：对缺失、异常或关键字段，仍回到 Excel 源报告确认。

## 7. 临床价值

对临床医生来说，这套系统的价值集中在四点：

1. 减少整理负担：从文件夹和 Excel 翻找，变成按患者索引进入。
2. 提高复核效率：设备身份、参数、事件和结论集中展示，关键字段可回源。
3. 支持随访质控：多次记录按时间聚合，方便发现资料缺失和随访连续性问题。
4. 保留原始依据：系统只是整理和展示，不切断原始报告这个临床源头。

## 8. 科研价值

当前版本尚不直接回答医学研究问题，但已经为后续研究准备了基础设施：

- 队列筛选：可按品牌、型号、植入日期、随访次数筛选样本。
- 字段字典：可围绕电池、导线、模式、事件和结论建立变量口径。
- 分析表导出：后续可以把患者档案转换成长表或宽表。
- 质量控制：审计结果可以帮助决定哪些字段适合进入统计分析。
- 建模准备：在人工确认变量口径后，再考虑机器学习或时间序列分析。

## 9. 边界与下一步

### 9.1 边界

- 不是诊断工具。
- 不替代医生阅读原始报告。
- 不改变任何临床判定阈值、参数规则或起搏器逻辑。
- 不在汇报材料中暴露患者姓名、登记号或单患者记录。

### 9.2 下一步

- 品牌归一：规范 Unknown、Vitatron 大小写等标签。
- 字段字典：明确字段含义、单位、来源区域和适用品牌。
- 脱敏演示包：生成可以对外演示的非真实患者数据集。
- 分析表导出：输出科研分析可用的长表/宽表。
- 随访质控视图：把缺失字段、异常格式和多次记录连续性做成质控入口。

## 10. PPT 转写原则

PPT 不应复述所有工程细节，而应把研究报告压缩成一条临床叙事：

散落报告很多 → 人工查找和复核慢 → 系统把报告整理成患者资料库 → 医生按患者看状态和历史 → 关键字段能回源 → 审计显示当前资料整理可靠 → 临床上减少整理负担，科研上形成可复用底座。
