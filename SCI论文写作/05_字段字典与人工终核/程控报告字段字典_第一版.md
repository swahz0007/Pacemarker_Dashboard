# 程控报告字段字典第一版

生成日期：2026-05-17  
用途：论文一和论文二的变量口径草案。

## 字段使用原则

- 优先使用程控报告内直接存在的字段。
- 不把外部临床结局纳入当前字段字典。
- 字段必须保留来源区域，便于回源复核。
- 单位、缺失值、适用设备类型需要在正式投稿前冻结。

## 患者与设备身份字段

| 字段 | 来源区域 | 用途 | 备注 |
|---|---|---|---|
| 登记号 | patient-level / header | 患者级合并 | 非标准或代理命名记录需单独标记 |
| 姓名 | header | 内部匹配 | 投稿和演示必须脱敏 |
| 性别 | header | 基本特征 | Unknown 单独统计 |
| 年龄（岁） | header | 基本特征 | 正式分析前转数值 |
| 品牌 | header | 分层分析 | 需统一 Vitatron 大小写和 Unknown |
| 型号 | header | 型号分布 | 可列前 10-15 型号 |
| 植入日期 | header | 随访起点 | 需清洗日期格式 |
| 设备类型 | 文件名/模板/型号 | 分层分析 | 正式投稿前需固定判定规则 |

## 人工终核策略

当前不做全量人工终核，也不人工收集临床结局。

如论文投稿或审稿需要，可做有限抽样终核：
- 优先终核品牌、型号、设备类型、植入日期、程控日期。
- 优先终核电池电压、预估寿命、右室阈值、右室阻抗、右室感知。
- 终核表记录 AI 提取值、原始报告值、人工终核值、是否一致、修正原因、复核人、复核日期。

人工终核的作用是验证关键字段可靠性，不是让人工补齐所有临床结局。
