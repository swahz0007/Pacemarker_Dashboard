# Pacemaker Dashboard SCI 写作工作区

本目录用于把 Pacemaker Dashboard 从一个前端看板项目，升级管理为一个持续增长的 CIED 程控真实世界数据库科研项目。

## 当前总纲

先把现有程控报告数据做成可审计的真实世界数据库，再用这个数据库连续产出方法学论文、临床回顾性论文、纵向随访论文和亚组论文。临床结局只在厂家、医院系统或信息科能够自动批量提供时再考虑，不人工逐个翻病历。

## 推荐阅读顺序

1. `00_项目总览与路线图/科研转化路线图.md`
2. `00_项目总览与路线图/课题成果归属矩阵.md`
3. `01_论文一_数据库构建与字段级审计/论文一_中文构思稿.md`
4. `02_论文二_三甲医院程控报告回顾性分析/论文二_中文构思稿.md`
5. `04_数据冻结与质控/数据冻结_20260517.md`

## 硬边界

- 当前阶段不人工收集死亡、住院、急诊、心衰恶化、再手术、导线修复等临床结局。
- 当前阶段只使用程控报告本身已有的信息：设备身份、程控日期、植入日期、参数、事件、结论、下次程控建议。
- 8000 元自筹课题只作为基础平台和前期研究，不把 SCI 写成硬结题指标。
- 后续每篇 SCI 尽量支撑一个更高级课题，避免一次性把成果全部消耗掉。
