# 前端 UI 规范与当前细节

## 1. 设计目标

- 医疗研究工具应稳定、清晰、克制，不追求装饰性效果。
- 正文和关键值优先可读，卡片仅用于分组，不应占据过多空白。
- 同一信息层级使用一致字号、间距、边框和颜色。
- 长文本必须换行，不得覆盖相邻元素。

## 2. 已完成的整体优化

- 提高卡片标题、标签、关键数值和正文的字号比例。
- 降低卡片冗余高度和内边距。
- 内容区最大宽度约束，超宽屏不无限拉伸。
- 总结卡全宽；电池与关键事件在桌面端并排。
- 统一浅色/暗色主题、阴影、圆角和边框。
- 添加 1200px、1800px 和移动端响应式规则。
- 患者项和历史记录头改为可访问按钮，支持焦点状态。
- 图表字号、点击区域和移动端高度已调整。
- 数据质量与人工核查入口放在总览页末尾，默认折叠，仅保留待核查数量提示；展开后查看完整统计和异常明细。
- 异常来源支持直接尝试打开原始文件，并保留复制路径作为兼容备用操作。
- 数据质量中心支持按处理状态筛选，逐条保存“待核查 / 已确认原报告无误 / 源文件已修正 / 暂时忽略”和核查备注；状态仅保存在本机浏览器，不修改原始 Excel 或 NAS。
- 患者详情页将核心结论与下次随访置于首要阅读卡，`[!]` 事件按 AMS、AT/AF、VT 和其他信息分组；普通事件保持中性层级，不根据原始文本自动判定风险。
- 患者详情页 `[B]` 电池与起搏参数按实际字段展示电压、预估寿命、模式和频率；不额外虚构“报告状态”字段。`[V]` 区域按实际存在的腔室展示阻抗、感知、阈值和输出，缺失值统一显示 `—`。
- 侧栏新增“字体与显示”设置，可用受控预设局部调整患者列表、参数标签/数值和核心结论的字号、字重、强调色及界面密度；设置只保存到本机浏览器。

## 3. 当前本地版本

资源标识：

- `style.css?v=318`
- `app.js?v=318`
- `identity-client.js?v=2`

正式线上仍为 `style.css?v=304`、`app.js?v=304`。

## 4. 关键事件遮挡问题

问题来源：

- `v304` 使用四列网格。
- `.event-item` 设置 `white-space: nowrap`。
- AT/AF、VT 描述较长时越过自身网格，覆盖后续内容。

本地 `v305` 修复：

- 改为两列 `minmax(0, 1fr)` 网格。
- 允许正常换行和 `overflow-wrap: anywhere`。
- 子项设置 `min-width: 0`。
- 不再把“已脱敏：自由文本不发布”拼接到事件摘要。
- 手机端继续保持单列。

## 5. UI 内容规则

- 卡片标题用短名词，如“关键事件”“导线状态”。
- 缺失值统一使用“无”“未指定”或 `--`，不要混用开发占位文字。
- 脱敏策略说明不应出现在医学数值旁边；应在页面说明或数据政策中表达。
- 结论与下一次随访在总结卡中分区显示。
- 长结论正常换行，不使用横向滚动。
