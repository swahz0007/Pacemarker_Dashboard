# 项目关键变更记录

## 2026-07-14

### I-001A 数据质量与人工核查中心

- 离线数据包新增质量摘要：扫描文件、匹配成功、模板未匹配、待核查问题和隔离记录。
- Dashboard 新增只读“数据质量与人工核查”区域，支持问题类型、严重级别和关键词筛选；后续调整为总览页末尾默认折叠，避免抢占临床查询主视线。
- 异常明细显示患者/登记号、文件名、问题说明、原始字段摘要和来源路径，并支持复制来源路径。
- 异常来源路径新增“打开文件”按钮，尝试调用 Windows 默认程序打开原始 Excel；复制路径作为浏览器拦截时的备用方式。
- 起搏模式图例恢复 Chart.js 原生展示与交互，不额外添加悬停动画。
- 全量处理遇到未匹配模板或提取不完整时，继续打开 Dashboard 展示核查中心，同时保留既有稳定患者输出。
- 公开演示构建不携带本地路径和核查明细。
- `npm run check:js`、Python 单元测试（10 项）和离线样本数据包检查通过。
- 当前状态：I-001A 本机通过，已转入 I-001B；等待本轮科室人工测试。

### I-001B 本机核查状态

- 数据质量中心增加“处理状态”筛选，以及逐条保存“待核查、已确认原报告无误、源文件已修正、暂时忽略”。
- 每条问题支持记录核查备注；选择“暂时忽略”时必须填写原因。
- 状态与备注仅保存在本机浏览器，不修改原始 Excel，不写入 NAS；来源文件打开/复制能力保持不变。
- 本机状态统计会同步显示待核查、已确认、已修正和暂时忽略数量。
- 当前状态：本机通过，待科室人工测试；确认后进入 I-001C。

### I-002 `[!] 事件与结论` 阅读层级

- 患者详情页将核心结论与建议下次随访提升为首要阅读卡，长文本完整换行。
- 关键事件改为独立全宽分组卡片，按 AMS、AT/AF、VT 和其他信息展示。
- 普通事件采用中性浅色层级，不根据原始文本猜测临床风险；保留原始内容和“无”状态。
- 当前状态：本机通过，待科室人工测试；确认后进入 I-003。

### I-003 `[B]`、`[V]` 参数重排

- `[B]` 区域拆分为实际字段中的电池电压、预估寿命、起搏模式和上下限频率，改为更易扫描的参数卡；移除未存在于模板中的“报告状态”展示项。
- `[V]` 区域按实际存在的腔室展示阻抗、感知、阈值和输出，缺失值显示 `—`，无数据时显示明确空状态。
- 历史成功报告进一步按“电池 / 起搏参数 / 阈值测试”分区，并在子区内按心房、右心室、左心室等腔室建立低对比度边界框，保留全部原始字段。
- 未增加临床阈值判断，不改变原始提取字段；本机通过，待科室人工测试。

### I-004 分区字体与显示设置

- 侧栏新增“字体与显示”入口，提供界面密度、患者列表字号、参数标签/数值字号、核心结论字号/字重/颜色的受控预设。
- 通过语义 CSS 变量隔离显示区域，局部调整不会整体放大页面；报告分区间距也支持紧凑、标准和宽松三档。
- 设置仅保存到当前浏览器本地，不写入患者数据、原始 Excel 或离线包配置；支持一键恢复默认。
- 当前状态：本机通过，待科室人工测试。
